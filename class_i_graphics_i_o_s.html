bool IGraphicsIOS::OSFindResource ( const char *  filename,
const char *  type,
WDL_String &  result 
)
overrideprotectedvirtual

Find the full, absolute path of a resource based on it's filename (e.g.

“background.png”) and type (e.g. “PNG”) On macOS resources are usually included inside the bundle resources folder. In that case you provide a filename and this method will return the absolute path to the resource. In some cases you may want to provide an absolute path to a file in a shared resources folder here (for example if you want to reduce the disk footprint of multiple bundles, such as when you have multiple plug-in formats installed). On Windows resources are usually baked into the binary via the resource compiler. In this case the filename argument is the resource id. The .rc file must include these ids, otherwise you may hit a runtime assertion. It is also possible to pass in an absolute path in order to share resources between binaries. Behind the scenes this method will make sure resources are loaded statically in memory.

Parameters
filenameThe resource filename including extension. If no resource is found the method will then check filename as if it is an absolute path.
